Google's mailing list really just doesn't work for me. I tried posting
the following today and as per usual I get an error.

All,

Please let's use the Type Modules Wiki page to develop the Data Space
aspects of SIOC. The convention should be as follows:

1. Check if an existing sioct:Container subclass is in place for desired
Data Space (if a Space is relevant to what you are adding) 2. Check if
an existing sioct:Item subclass is in place for the desired "Items" of
Item 1 (this is where you expose Objects/Entities associated with the
Schema / Ontology that you are hooking into SIOC).

Edit the following page according:
http://wiki.sioc-project.org/w/TypesModule

From my vantage point, Wikiwords are currently covered, but following
the guide above, we can certainly inject Objects with more specificity
to the Wiki domain into SIOC .

Kingsley

This is interesting indeed. Its author has recently joined SIOC-Dev
and hopefully will give some more insights.

There are relations between WikiRdf and the SIOC Types module, some of
which are already described in [3]. Since a sioc_t:WikiArticle is also
a subclass of sioc:Post there may be more SIOC properties that can be
of use here.

P.S. I had some more thoughts about representing wikis in SIOC but
that was left on a paper until there's time to work on that. It's
great that Earle Martin is moving fast ahead with WikiRdf as
describing wikis in RDF is an important field.
